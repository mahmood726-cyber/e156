# E156 Micro-Paper Standard — Design Specification

**Date**: 2026-03-26
**Status**: Approved
**Location**: `C:\E156\`

---

## 1. Overview

E156 is a compact, executable micro-paper standard for meta-analyses and evidence syntheses. The project consists of:

1. **The Specification** — a format standard defining a 7-sentence, 156-word body with an outside note block
2. **An Interactive HTML Tool** — a single-file browser app that showcases the format and provides a composer/validator

The HTML tool uses a **two-mode design**: an editorial showcase page (Magazine mode) that explains and demonstrates E156, and a focused writing environment (Workshop mode) for composing and validating micro-papers.

**Visual direction**: NYT long-read editorial style — serif typography, generous whitespace, restrained palette, centered column layout.

---

## 2. The E156 Format

### 2.1 Core Body Rules

- Exactly 7 sentences
- Exactly 156 words in final release mode
- Single paragraph, no title, no headings, no citations, no references, no links, no metadata inside the body

### 2.2 Sentence Schema

| # | Role | Description |
|---|------|-------------|
| 1 | Question | Population/condition, intervention/exposure/comparison, main endpoint |
| 2 | Dataset | Studies, participants, scope, follow-up |
| 3 | Method | Synthesis method and analytic level |
| 4 | Primary result | One main estimate with interval |
| 5 | Robustness | Heterogeneity, sensitivity, subgroup, consistency, or stability |
| 6 | Interpretation | Plain-language meaning with restrained wording |
| 7 | Boundary | Limitation, scope limit, or caution |

### 2.3 Outside Note Block

Fields (all outside the body):
- App (link to interactive tool)
- Data (link to dataset)
- Code (link to repository)
- DOI
- Version
- Date
- Validation status (optional)

### 2.4 House Style

- One idea per sentence
- Numbers over adjectives
- No hype
- No causal overreach unless justified by design and evidence
- Limitation mandatory
- Paragraph must make sense when screenshot alone

## 3. Interactive HTML Tool — Architecture

### 3.1 Two-Mode Design

**Magazine Mode** (default): Editorial showcase page. Explains E156, renders example micro-papers, ends with a call-to-action to enter Workshop mode.

**Workshop Mode**: Full-screen composer with live validation sidebar. Accessed via "Write one" button or `#compose` URL hash. "Back to E156" link returns to Magazine mode.

Transition: CSS fade. URL hash (`#compose`) makes modes bookmarkable.

### 3.2 Magazine Mode — Content Flow

1. **Masthead**: "E156" in large serif, subtitle "The micro-paper standard" in lighter weight. Minimal — no logo, no nav bar.

2. **The Argument** (~150 words): Why 156 words? Why 7 sentences? Meta-analyses produce thousands of pages but the decision-maker needs one paragraph. E156 is that paragraph, with rules to keep it honest.

3. **Example Micro-Papers** (2-3): Each rendered as a typographic block (thin left border, pull-quote style). Below each, a subtle annotation layer showing the 7 sentence roles in muted color. Examples:
   - Fragility Atlas (observational/multiverse analysis)
   - MetaRep (novel replication probability method)
   - A clinical trial meta-analysis (cardiology domain)

4. **The Rules**: Sentence schema as a clean numbered list. House style rules as a short paragraph.

5. **Call to Action**: Single button — "Write one →" — transitions to Workshop mode.

### 3.3 Workshop Mode — Layout

**Desktop**: Two-panel, side by side. Editor (60% width) + Validation (40% width).
**Mobile**: Stacked — editor on top, validation below.

#### Left Panel — The Editor

- **Body textarea**: Large serif font matching showcase. Placeholder shows sentence schema as ghost text. Auto-expanding height.
- **Outside Note Block**: Below textarea. 7 input fields (App, Data, Code, DOI, Version, Date, Validation Status). Collapsed by default with "Add outside note block" toggle.

#### Right Panel — Live Validation

- **Word counter**: Large number display. Green at exactly 156, red otherwise. Format: `143/156`.
- **Sentence counter**: Same treatment. Green at 7.
- **Sentence Role Map**: Each detected sentence gets a colored positional label (S1: Question, S2: Dataset, etc.). Flags missing or extra sentences.
- **House Style Checks** (live):
  - No headings detected
  - No links detected
  - No citations detected
  - Result sentence contains number + interval
  - Limitation sentence present (S7 heuristic)
- **Overall Status Badge**: "VALID" (green), "DRAFT" (amber), or specific issues listed.

**Validation timing**: Debounced at 300ms after typing stops. No intrusive popups.

### 3.4 Export

Export bar below editor in Workshop mode. Four buttons, NYT-styled:

1. **Copy Body** — clipboard copy of the 156-word paragraph. Toast notification "Copied."
2. **Download .md** — body paragraph + `---` separator + outside note block as definition list.
3. **Download .json** — structured object:
   ```json
   {
     "body": "...",
     "sentences": ["S1...", "S2...", ...],
     "wordCount": 156,
     "sentenceCount": 7,
     "outsideNote": {
       "app": "...",
       "data": "...",
       "code": "...",
       "doi": "...",
       "version": "...",
       "date": "...",
       "validation": "..."
     },
     "meta": {
       "created": "2026-03-26T...",
       "valid": true
     }
   }
   ```
4. **Print View** — triggers browser print dialog. Print CSS renders body + outside note block on one clean page at ~14pt serif, centered. All UI chrome hidden.

### 3.5 localStorage Persistence

- Key: `e156_draft` (unique to this app)
- Saves: body text + outside note block field values
- Auto-restores on page load
- Cleared on successful export (with confirmation)

### 3.6 Sentence Detection

Split on `.` followed by space or end-of-string, with guards for common abbreviations:
- Statistical: "vs.", "CI.", "95%.", "p.", "e.g.", "i.e."
- Academic: "et al.", "Fig.", "No."
- Numeric: decimal numbers (e.g., "0.85")

## 4. Technical Decisions

| Decision | Choice | Rationale |
|----------|--------|-----------|
| File structure | Single HTML file, all CSS+JS inline | Per html-apps pattern. No build step, no dependencies. |
| Framework | Vanilla JS | Simple DOM manipulation sufficient. No reactivity framework needed. |
| Typography | Georgia / Charter / Times New Roman stack | NYT editorial feel. System fonts, no external loads. |
| Color palette | Off-white bg (#faf9f6), near-black text (#1a1a1a) | High contrast, warm editorial tone. |
| Column width | Max ~680px centered | Standard long-read article width (NYT, Medium). |
| Line height | 1.7–1.8 | Generous for readability at serif sizes. |
| Examples | Embedded as `<script type="application/json">` | Self-contained single file. Also kept as separate .json files for pipeline use. |
| PRNG | Not used | No sampling or randomization needed. |
| Math.random() | Avoided | Per project rules, though not applicable here. |

## 6. Production Workflow (per spec)

The E156 spec defines a production workflow for writing micro-papers:

1. Draft at 165–180 words
2. Reduce to 156 words
3. Run multi-persona review (Clinician, Statistician, Methods Editor, Skeptic, Reader)
4. Freeze body
5. Attach outside note block

The HTML tool supports steps 1–2 and 5 directly. Step 3 is handled externally (e.g., Claude multi-persona review skill). Step 4 is a human decision.

---

## 7. Validation Checklist (built into tool)

- [ ] 7 sentences
- [ ] 156 words
- [ ] Sentence roles preserved (positional)
- [ ] Result sentence (S4) contains estimate and interval
- [ ] Limitation sentence (S7) present
- [ ] Interpretation (S6) does not contain causal language unless warranted
- [ ] Outside note block present and populated
- [ ] No headings, links, citations, or metadata in body
